Becoming a rustling

Frank Eschner - 22 Dec 2022

A few months ago, a friend of mine told me that he was trying to drive an Arduino with rust. "Why would you want to do that?" I asked. "For no particular reason" was his answer, as he looked at me like this question was weird.

TL;DR: Just show me what you programmed in rust.

I did not know it yet, but on the evening of this very day, I started my rust journey. Thinking about my question, I now do understand why it was a bit off. Rust is a beautiful and refreshing language to learn, that allows you to write blazingly fast code, gives you precise and good feedback about what you do wrong, and last but not least, grows your trust in the resilience of the code you write.

After learning some basics, I set myself the goal of re-programming the baby neural network I wrote in python some time ago. Coming from python, I was used to a wide variety of modules that drastically reduce the amount of actual development one has to do. In rust, these modules are called crates, but there are of course way fewer (in variety and number), than what I was used to. After stumbling around and between several crates, I got to the point of asking myself, what I came here for in the first place.

Sticks and stones may break my bones, but learning something from scratch will never hurt me

So off I was. First things first. I needed some data for my (yet, non-existing) neural network to learn. Remember this image?

Creating this XNOR dataset is pretty straightforward. Sample from four different 2D Gaussian distributions that have the same variance but are just shifted along one or both dimensions. Afterward, group the ones on opposite sites. One group contains the positive, and the other group the negative samples.

Unfortunately, rust does (natively) not provide the possibility of sampling from a Gaussian distribution. It just allows you to sample from Uniform distribution. So after a couple of minutes of remembering previous lectures about sampling, I got to the conclusion of using the Box-MÃ¼ller method to implement my very first struct. 

pub struct Gaussian {
    mean: f64,
    std: f64,
    random_number_generator: ThreadRng
}

impl Gaussian {
    pub fn new(mean: f64, std: f64) -> Gaussian {
        Gaussian { mean: mean, std: std, random_number_generator: thread_rng() }
    }
    
    ...
    
    fn box_muller(&mut self) -> Vec<f64> {
        let x_1: f64 = self.random_number_generator.gen::<f64>();
        let x_2: f64 = self.random_number_generator.gen::<f64>();

        let mut g: Vec<f64> = Vec::new();
        
        let y: f64 = f64::sqrt(-2. * f64::ln(x_1)) * f64::cos(2. * std::f64::consts::PI * x_2);
        let z: f64 = f64::sqrt(-2. * f64::ln(x_1)) * f64::sin(2. * std::f64::consts::PI * x_2);
        g.push(self.mean + self.std * y);
        g.push(self.mean + self.std * z);
        g
    }
}

Adding a simple public samples method that pays attention to the fact that the previously mentioned sampling method returns two samples at a time, I was now able to sample from a Gaussian distribution. Happy about how smooth that implementation went, I started my second struct in that project, containing the actual XNOR dataset.

pub struct XnorDataset {
    pub x: Vector2D,
    pub y: Vector2D,
}

impl XnorDataset {
    pub fn new(num_samples: usize) -> XnorDataset {
        let mut g_1: Gaussian = Gaussian::new(0., 0.2);
        let mut g_2: Gaussian = Gaussian::new(1., 0.2);

        let mut x_values: Vec<f64> = Vec::new();
        let mut y_values: Vec<f64> = Vec::new();

        x_values.append(&mut g_1.samples(num_samples/2));
        x_values.append(&mut g_2.samples(num_samples/2));
        x_values.append(&mut g_1.samples(num_samples/4));
        x_values.append(&mut g_2.samples(num_samples/4));
        x_values.append(&mut g_1.samples(num_samples/4));
        x_values.append(&mut g_2.samples(num_samples/4));

        y_values.append(&mut vec![1.; num_samples/4]);
        y_values.append(&mut vec![0.; num_samples/2]);
        y_values.append(&mut vec![1.; num_samples/4]);

        let x: Vector2D = Vector2D::new(x_values, [2, num_samples]).transpose();
        let y: Vector2D = Vector2D::new(y_values, [num_samples, 1]);

        XnorDataset { x, y } 
    }
}

Here I just use two instances of the previously introduced Gaussian struct, one with a mean of 0 and one with a mean of 1. Both have the same standard distribution of 0.2. Afterward, I created two vectors (for x and y values) and appended data from both distributions. Finally, these (1D) vectors together with the desired shape, were used to initialize two instances of the Vector2D struct. This struct allows you to create 2D Vectors (some people might just call this a matrix) and do things like dot products, element-wise addition/subtraction, etc.

At this point, we have our dataset ready in the form of Vector2D instances (matrices) and we can perform all the mathematical operations we need... nearly. Truth is, there is no Vector2D struct in (native) rust. We need to create it ourselves. Long story short: this little de-tour was the major challenge in this project. Even without all the testing (which was actually a lot of fun to write), around 60% of the whole code base (~600 lines) belongs to the Vector2D struct and all its implementations (doing the desired mathematical operations). Here are just some snippets I would like to present to you. First, the actual struct definition. Secondly, the implemented ability to get values from a specific row & column. Lastly, the code that is responsible for transposing.

pub struct Vector2D {
    pub values: Vec<f64>,
    pub shape: [usize; 2]
}

impl Vector2D {
    pub fn new(values: Vec<f64>, shape: [usize; 2]) -> Vector2D {
        if values.len() != shape[0] * shape[1] {
            panic!("Shape {:?} does not match length {}", shape, values.len());
        }
        Vector2D { values, shape }
    }

    ... 
    
    pub fn get_mat_value(&self, i: usize, j: usize) -> &f64 {
        if (i >= self.shape[0]) | (j >= self.shape[1]) {
            panic!("Index out of bounds.)
        } else {
            &self.values[i * self.shape[1] + j]
        }
    }
    
    ...
    
    pub fn transpose(&self) -> Vector2D {
        let new_shape: [usize; 2] = [self.shape[1], self.shape[0]];
        let mut new_values: Vec<f64> = vec![];

        for row in 0..new_shape[0] {
            for column in 0..new_shape[1] {
                new_values.push(*self.get_mat_value(column, row));
            }
        }
        Vector2D::new(new_values, new_shape)    
    }
}

Putting it all together in the main rust file, the whole process looks like this. First, initialize the dataset as well as the neural network itself. Then start the training process. Note that I use the very same architecture, number of iterations, and learning rate, as in my python counterpart.

fn main() {  
    let data: XnorDataset = XnorDataset::new(200);
    let mut nn = NeuralNetwork::new(vec![2, 3, 1]);
    
    nn.training(data.x, data.y, 1000, false);

    println!("\n---------------- Post-training parameters ----------------");
    for n in 0..nn.parameters.weights.len() {
        println!("\nWeights[{}]: ", n);
        nn.parameters.weights[n].print();

        println!("\nBias[{}]: ", n); 
        nn.parameters.biases[n].print();
    }
}
