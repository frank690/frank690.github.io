Most of my projects are either programming, working with electrical circuits, or handcraft (usually metalwork) related. But oftentimes these lines blur and it is a mix of the above. In the following, you will find an (incomplete) list of my present and past projects. Some of them are done, some of them will soon be and some will probably never be done. If I have some past projects worth sharing and I find the time and muse to write everything down, you will find a post on this blog.

MQTT-Forger

I came up with this while getting to know MQTT and playing around with some IoT-Systems. Said system's task was to detect anomalies in the data stream they received, but creating such a stream was not an easy task at this point. So I decided to write my own program that could do just that.

Dependency-Graph

If you have a big python repository (possibly even a monorepo), you should be keen to keep all the imports inside the contained modules well structured. Otherwise, you may end up creating an inflexible mess that will create a lot of technical debt for you in the future. Analyzing such dependencies with a graph has been a big help for me.

Graden-of-Things

This is a project about my Garden-of-Things sensors. If you want to keep track of the humidity, temperature, sun intensity, etc. of your plants all day and night and might even like to solder your own sensor for that, you have come to the right address.

MedForm

Is a coop project with a good friend of mine. An android app + a python backend to gather medical data from patient surveys in a nice and easy fashion. At this point in time, this project is not open-source. If you are still interested, drop me a line via mail.
